## Actualización: Resolución de Bug (06/03/2026)
*   **Problema:** Migración 0002 corrupta con valor por defecto inválido (`default=1`) en campo `DateTimeField`. Esto impedía el inicio de Django y el acceso al Admin.
*   **Solución Aplicada:**
    1.  Eliminación física del archivo de migración corrupto.
    2.  Estabilización temporal del modelo comentando el campo `created_at`.
    3.  Limpieza de datos de prueba desde la shell de Django (`Keyword.objects.all().delete()`).
    4.  Implementación de migración en dos fases (campo opcional -> campo definitivo) para evitar bloqueos en consola.
    5.  Uso de `--fake` para sincronizar el estado físico de la base de datos con la historia de Django.
*   **Resultado:** Bug resuelto. La tabla `Keyword` ahora incluye `created_at` funcional con `auto_now_add=True`. Fase 1 validada y cerrada.
