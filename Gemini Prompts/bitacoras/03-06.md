### 2. Reflexiones sobre Extracción y Eficiencia
Durante esta sesión, realizamos un análisis comparativo entre la **API REST** y el **Scraping Web** (basado en el MVP anterior):
*   **El Dilema de la Calidad vs. Tiempo:** 
    *   La API es extremadamente eficiente para el descubrimiento masivo (listados de +1000 licitaciones), pero entrega información "pobre" (faltan criterios de evaluación y anexos).
    *   El Scraping (Selenium) permite obtener la ficha profunda ("Resumen de bases"), pero es inviable para procesos masivos debido a la latencia y fragilidad.
*   **Arquitectura de "Embudo" propuesto:** 
    1.  **Capa 1 (Radar):** Uso de la API para filtrar títulos en milisegundos.
    2.  **Capa 2 (Lupa):** Uso de la API para obtener descripciones y rubros de los candidatos.
    3.  **Capa 3 (Análisis Profundo):** Reserva del Scraper/Analista solo para licitaciones pre-seleccionadas.

### 3. Descubrimientos Técnicos
*   **Normalización de Texto:** Implementada función `normalize_text` (basada en `unicodedata`) para ignorar tildes, mayúsculas y variaciones lingüísticas (ej: "Médico" = "medico"), aumentando drásticamente la precisión del filtrado.
*   **Gap de Datos:** Confirmamos que la API no entrega los campos de "Criterios de Evaluación" ni "Requisitos técnicos", los cuales son vitales para el análisis de rentabilidad por IA.
